# Описание проекта

Разрабатываемое приложение представляет собой облачную веб-платформу для проведения образовательных и логопедических занятий.

Система позволяет:
- проводить видеозвонки;
- хранить учебные материалы;
- загружать изображения, документы и видео;
- вести чат между учениками и преподавателями;
- отправлять уведомления;
- осуществлять оплату занятий;
- отслеживать прогресс учеников.

---

# Пользователи системы

| Роль | Возможности |
|---|---|
| Ученик | посещение занятий, просмотр материалов, чат |
| Логопед/преподаватель | проведение уроков, загрузка материалов, проверка прогресса |
| Администратор | управление системой и пользователями |

---

# Основной функционал

- регистрация пользователей;
- авторизация;
- проведение видеозвонков;

- поиск материалов;

# Этапы развития проекта

## MVP

### Предполагаемая нагрузка
До 20 пользователей.

### Используемые сервисы

| Сервис | Назначение |
|---|---|
| Cloud Run | backend приложения |
| Cloud SQL | PostgreSQL база данных |
| Cloud Storage | хранение файлов и видео |
| Artifact Registry | хранение Docker образов |

### Причины выбора

На начальном этапе выбраны serverless-решения для минимизации расходов и упрощения администрирования инфраструктуры.
Cloud Run автоматически масштабируется и не требует поддержки виртуальных машин.
Cloud SQL используется для хранения данных пользователей, уроков и платежей.
Cloud Storage применяется для хранения видеоуроков, изображений и документов.

# Причины масштабирования

При увеличении количества пользователей возрастает нагрузка на backend и систему хранения файлов.
CDN уменьшает задержки при доставке видеофайлов и снижает нагрузку на backend.
Load Balancer обеспечивает стабильное распределение запросов между экземплярами приложения.
Secret Manager используется для безопасного хранения API ключей и токенов.
Cloud Monitoring позволяет отслеживать ошибки и нагрузку на инфраструктуру.

# Prod

## Предполагаемая нагрузка
До 500 пользователей.

---

# Используемые сервисы

| Сервис | Назначение |
|---|---|
| Memorystore Redis | кеширование |
| Pub/Sub | уведомления |
| Cloud SQL HA | отказоустойчивая база данных |
| VPC | изоляция сети |
| Backup | резервное копирование |
| Logging | логирование |

---

# Причины выбора сервисов

Redis используется для кеширования данных и ускорения работы приложения.
Pub/Sub применяется для отправки уведомлений через Telegram, VK, MAX и email.
Cloud SQL HA обеспечивает отказоустойчивость базы данных.
VPC используется для повышения безопасности сети.
Резервное копирование необходимо для сохранности пользовательских данных.
Cloud Logging позволяет анализировать ошибки и состояние системы.

# Масштабирование

Архитектура проекта предусматривает постепенное масштабирование инфраструктуры в зависимости от роста нагрузки.
На этапе MVP используются минимальные serverless-ресурсы для снижения стоимости.
На этапе тестирования добавляются сервисы балансировки нагрузки и мониторинга.
В production используется отказоустойчивая инфраструктура с кешированием, CDN и системой уведомлений.
