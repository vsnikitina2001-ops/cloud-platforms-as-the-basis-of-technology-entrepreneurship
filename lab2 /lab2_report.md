### 5. Изменение порта

В конфигурации сервиса был изменен порт с дефолтного (8080) на **8090**.

**Результат:**
- Сервис перестал корректно отвечать
- Возникает ошибка при обращении

**Причина:**
Cloud Run ожидает, что контейнер слушает порт, указанный в переменной среды `PORT`.  
Если приложение не настроено на новый порт — сервис становится недоступным.

### 6. Работа с ревизиями и трафиком

Были созданы несколько ревизий сервиса:

- Первая — рабочая (порт 8080)
- Вторая — с измененным портом (8090)

Выполнено переключение трафика между версиями:

- 100% на рабочую версию → сервис работает  
- 100% на новую версию → сервис не работает  
- Разделение трафика (например 50/50)

**Вывод:**
Cloud Run позволяет гибко управлять версиями и постепенно выкатывать изменения.
